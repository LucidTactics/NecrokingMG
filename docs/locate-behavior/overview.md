# Overview — behavior routing map

Entry point for the **locate-behavior** skill. Match the request to an area below, then
open the listed `<area>.md`. Areas marked **(not yet documented)** have no doc yet —
**research them and add the doc** per README.md → "Self-healing", then update this file.

> Hints for undocumented areas are **tentative** (inferred from folder/namespace names,
> not yet verified). Don't trust them as fact — confirm by reading the code, and capture
> what you learn in a new `<area>.md`.

## Behavior → area quick index

Use this to pick a starting area. When the routed area isn't documented yet, document it.

- **A spell does the wrong thing / adding a spell** → game1-partials.md (`Game1.Spells.cs`) + `Game/` (SpellCasting, SpellEffectSystem) + `Data/Registries/` (SpellRegistry). See also `docs/spells.md`. **Summon-spell / `spells.json` field schema (incl. horde `summonQuantity`, `summonTargetReq`/`spawnLocation` enums, horde-cap clamp)** → [json-spells.md](json-spells.md).
- **"Can this spell be cast right now" — mana / cooldown / path-requirement cast gates, `CastResult`, castability predicates for HUD greying** → `Game/SpellCasting.cs` `SpellCaster.TryStartSpellCast` = the ONE canonical gate sequence (cooldown → `SpellDef.MeetsPathRequirements` = `CastResult.MissingPath` → `caster.Mana < SpellDef.EffectiveManaCost` = `NotEnoughMana`; targeting/range after). It MUTATES on success (deducts mana, stamps cooldown, writes `outPending`) — never call it as a per-frame query. Read-only helpers on `SpellCaster`: `HasSpellRequirements` (path gate ONLY, deliberately no mana/cooldown — filters the grimoire) + `DescribeMissingPath`; path levels = `Game/BuffSystem.cs` `EffectivePathLevel`. Gotcha: `UI/HUDRenderer.cs` `DrawSpellBar`'s low-mana overlay compares flat `spell.ManaCost`, not the path-discounted `EffectiveManaCost` the gate uses.
- **Spell tooltips & cost/mastery display — spellbar hover tooltip, grimoire tile cost/path badges, the path-mastery mana-discount formula** → spellbar tooltip = `UI/HUDRenderer.cs` `DrawSpellSlotTooltip` (fed by `_hoverSlotSpell` captured in `DrawSpellBar`, rendered via `Game1.Tooltips.RequestLines`; shows flat `ManaCost`, NOT the discounted cost the gate uses). The grimoire has **no hover tooltip** — spell tiles are widget-bound in `UI/GrimoirePanel.cs` `BindTile` (`cost_v` = flat ManaCost + fatigue icon, `path_v`/`path_i` = primary req level+icon); overlay/click routing = `UI/GrimoireOverlay.cs` (`Layout`/`ResolveClick`/`HitChild("tileN")`). Discount formula = `SpellDef.EffectiveManaCost` in `Data/Registries/SpellRegistry.cs` (`ManaCost / (casterLevel − PrimaryLevel + 1)`); its ONLY consumers: `Game/SpellCasting.cs` gate+deduct+`RefundSpellCast`, `AI/CasterUnitHandler.cs`, `DrawSpellBar` low-mana grey. Effect-time damage/range/aoe reads = `Game/SpellEffectSystem.cs` `Execute`/`SpawnProjectile`/`ExecuteStrikeFrom`/`ExecuteCloud` (SpellDefs are shared registry objects — never mutate them per-cast; effect-time path-level precedent = `Game/SpellPenetration.cs`).
- **Cast begin/end lifecycle, "is the necromancer casting", cast-time/channel state, gate/lock something for the cast duration (e.g. stop moving while casting)** → game1-partials.md (`_pendingCastAnim` field: set in `Game1.Spells.cs` `DispatchSpellCast`, cleared in `Game1.Animation.cs` `UpdateAnimations`/`UpdateChanneledCast`) + [movement.md](movement.md) (player movement gate = `Simulation.UpdateAI` `PlayerControlled` case, `PreferredVel = _necroMoveInput * speed`, with the `CorpseInteractPhase` zero-`PreferredVel` precedent).
- **Armed cursor mode / cast-target preview — a mode where the cursor carries a world preview (build-placement ghost) and the next world click executes it; where a spell targeting circle / AOE cursor overlay goes** → `UI/BuildingMenuUI.cs` (`IsPlacementActive` + `DrawGhostPreview` → `Render/EnvGhostRenderer.cs`, drawn from its PanelLayer `.WithDraw` in `Game1.cs`) + `Game1.WorldClicks.cs` `HandleWorldLeftClick` step 1 ("placement mode owns the click" — THE armed-mode click-intercept precedent) + ground-plane ring/marker drawing = `GameRenderer.Units.cs` `DrawHoverGroundMarkers`/`DrawEllipseOutline` (submitted at `WorldLayer.HoverMarkers` via `_cbHoverMarkers` in `CollectWorldItems`; project center + `center+(r,0)` with `WorldToScreen`, squash Y by `HoverMarkerFlatten` for the iso ellipse). No spell aim/armed state exists yet — slot keypress casts instantly (`Game1.cs` Update cast loop → `DispatchSpellCast(spellId, necroIdx, slot, mouseWorld)`).
- **Corpses — data model, picking up / carrying a corpse, click/F-key corpse interaction, corpse piles** → [corpses.md](corpses.md) (`Game/Simulation.cs` `Corpse`, `Game/CorpseInteractionManager.cs`, `Game1.cs` input + `_hoveredCorpseIdx`, `Game/ForagableSystem.cs` for the gather pattern).
- **Workers / jobs — assigning workers, job board, grave roster, auto-dispatch** → [jobs-workers.md](jobs-workers.md) (`UI/JobBoardUI.cs`, `UI/GraveRosterUI.cs`, `Game/Jobs/WorkerSystem.cs`, `AI/WorkerHandler.cs`).
- **Animation — unit anim state, who sets/changes it, dodge/block/flinch/death overrides, "unit slides while playing the wrong anim", anim priority/locks, cast-phase timing, attack anims** → [animation.md](animation.md) (`Render/AnimController.cs` `AnimState` + policy tables, `Render/AnimResolver.cs` `SetOverride`/`Resolve`, `Game1.Animation.cs` `UpdateAnimations` + the writer census) + game1-partials.md for the channeled-cast tick.

   [game1-partials.md](game1-partials.md) format.
2. Move its row to "Documented areas", mark the doc ✅ here, and tighten its behavior→area
   entries above with the real file names you found.
3. Commit the new doc (the skill is shared).
